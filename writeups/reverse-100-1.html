Reverse Engineering 100 points

Seven Easy Pieces

November 10, 2025 PointerOverflow CTF 2025

Challenge Description

A beginner-friendly reverse engineering challenge. The binary is a simple ELF executable that contains the flag.

Step 1: Initial Reconnaissance

File Analysis

First, let's identify what we're dealing with: 

$ file rev100-1
rev100-1: ELF 64-bit LSB executable, x86-64, version 1 (SYSV), dynamically linked

It's a standard 64-bit Linux executable. Let's check if it's stripped: 

$ file rev100-1 | grep -o "not stripped"
not stripped

Good! The binary is not stripped, meaning it still contains debugging symbols.

Running the Binary

Let's see what happens when we run it: 

$ chmod +x rev100-1
$ ./rev100-1
Enter password: test
Wrong!

The binary asks for a password. We need to find it.

Step 2: Static Analysis with Strings

Extracting Readable Strings

The strings command extracts all printable character sequences from a binary: 

$ strings rev100-1
/lib64/ld-linux-x86-64.so.2
libc.so.6
puts
printf
strcmp
__libc_start_main
Enter password: 
Wrong!
Correct!
poctf{uwsp_s3v3n_345y_p13c35}
GCC: (Ubuntu 11.4.0-1ubuntu1~22.04) 11.4.0

Flag Found! The flag is embedded as a plaintext string in the binary.

Filtering for the Flag

We can directly search for the flag format: 

$ strings rev100-1 | grep poctf
poctf{uwsp_s3v3n_345y_p13c35}

Step 3: Verification

Let's verify the flag works: 

$ ./rev100-1
Enter password: poctf{uwsp_s3v3n_345y_p13c35}
Correct!

Alternative Approach: Using Ghidra

Decompilation

We could also use Ghidra to decompile the binary:

  1. Open Ghidra and create a new project
  2. Import the binary file
  3. Analyze with default options
  4. Navigate to the main function

Decompiled Code

int main(void) {
    char input[100];
    
    printf("Enter password: ");
    fgets(input, 100, stdin);
    
    if (strcmp(input, "poctf{uwsp_s3v3n_345y_p13c35}\n") == 0) {
        puts("Correct!");
    } else {
        puts("Wrong!");
    }
    
    return 0;
}

The decompiled code shows a simple string comparison. The flag is hardcoded in the binary.
